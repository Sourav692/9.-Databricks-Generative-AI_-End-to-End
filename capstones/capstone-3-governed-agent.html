Ship a Governed, Monitored Agent

Turn the Unity Airways knowledge base into a tool-using assistant that answers policy and live-flight questions — then do the part a POC skips: deploy it, wrap it in AI Gateway guardrails, and watch it in production. The agent is the brain; the graded work is the ring around it.

01 · The brief

From answering bot to shipped, governed agent

Unity Airways already has a policy Q&A bot — the RAG chain from C1, evaluated and versioned in C2. Support leadership wants it to do more, safely, and in production.

  • One assistant, two kinds of question. Customers ask "UA118 tonight — delayed? and if it's cancelled, can I get a refund?" in one breath. That needs a document lookup (policy) and a live record lookup (flight status), chosen turn by turn — a tool-using agent, not a fixed chain.
  • Safe to put in front of customers. No leaked PII, no unsafe replies, a hard request-rate cap, and controlled spend — the exact list a security review reads out.
  • Observable in production. Quality dips and latency spikes should surface on a dashboard the same day, not arrive as a customer complaint a week later.
  • A loop that improves it. Questions the assistant fumbles should feed back into the C2 evaluation set so the next version is measurably better.
📌 ImportantThis capstone extends C1/C2 — it does not rebuild them. The RAG retrieval and the evaluation harness already exist. You wrap the C1 chain as one tool inside a larger agent, add a second (structured) tool, then spend most of your effort on deploy → govern → monitor — the production ring the earlier capstones stop short of.
02 · Explore

Scenario, architecture, milestones, rubric

What you'll build. Pick one path to the same governed outcome — both are graded on the same rubric.

Path A — custom ResponsesAgent

A tool-using agent authored on ResponsesAgent (from mlflow.pyfunc) that wraps the C1 RAG chain as a retriever tool, adds a get_flight_status UC-function tool, and optionally a managed MCP tool.

Register as unity_airways.rag.ua_support_agent, deploy with agents.deploy(...), then govern + monitor.

Path B — low-code (Agent Bricks + App)

An Agent Bricks Knowledge Assistant over the same index/docs, optionally routed by a Multi-Agent Supervisor that also reaches a Genie Agent for flight status, fronted by a Databricks App.

Same governance and monitoring requirements apply.

Both paths end at the same place: a governed serving endpoint behind AI Gateway guardrails, with a monitoring dashboard and a Review App feedback loop.

03 · Deliverables

Hand in all six

  • A deployed agent serving endpoint (Path A) or App URL (Path B) answering policy and flight-status questions, plus the Review App URL.
  • AI Gateway configuration — guardrails, rate limits, provider fallbacks, payload logging (config export or annotated screenshots).
  • A guardrail / responsible-AI policy — PII and safety stance, service-principal identity, UC grants, a filled RAI checklist.
  • An AI/BI monitoring dashboard showing operational + quality + business metrics over live traffic.
  • A short reliability & safety report (1–2 pp): uptime, p50/p95 latency, guardrail-hit rate, scorer trend vs the C2 baseline, one seeded-regression incident with the rollback (repoint @champion), open risks.
  • The updated evaluation set — the C2 dataset grown with production examples, plus the re-run scorecard.

05 · Field notes

What to remember on this build

📌 ImportantThe graded difficulty is the ring, not the brain. Wrapping the C1 chain as a tool and assembling a ResponsesAgent is a day of work. The endpoint that stays safe under a bad prompt, the dashboard that catches a regression before a customer does, and the loop that feeds fixes back into eval — that is where a governed agent is won or lost.
💡 Tip · fieldSet @champion on the deployed version and standardize the rollback drill: a regression is one set_registered_model_alias(...) away from resolved, with no client change — the same move from 06.5 / C2. Rehearse it once during M5 so the incident in your reliability report is a controlled demo, not a fire.
