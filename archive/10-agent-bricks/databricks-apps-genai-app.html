A GenAI App on Databricks Apps

Put a real front-end on your agent: a chat UI hosted on Databricks' serverless platform, governed by Unity Catalog, talking to your Model Serving endpoint.

01

The big idea

Databricks Apps hosts an interactive web app — a chat UI, a dashboard — directly on Databricks, on serverless compute, with no separate infrastructure.

  • Build with Python (Streamlit · Dash · Gradio) or Node.js (React · Angular · Svelte · Express).
  • An app = your code + an app.yaml; it declares resources (e.g. a Model Serving endpoint) and runs as an app service principal when deployed.
  • For GenAI it's a thin client over your agent/endpoint — keep the logic in the endpoint.
02

Build & deploy

The app is a thin UI; the agent/LLM logic stays in the endpoint. The app SP must be granted access.
# app.py — minimal Streamlit chat over a Databricks serving/agent endpoint
import streamlit as st
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.serving import ChatMessage, ChatMessageRole

w = WorkspaceClient()                 # deployed app: auto-auths as the app service principal
ENDPOINT = "unity-airways-agent"

st.title("Unity Airways Assistant")
st.session_state.setdefault("history", [])
for m in st.session_state.history:
    st.chat_message(m["role"]).write(m["content"])

if q := st.chat_input("Ask about bookings, baggage, refunds..."):
    st.session_state.history.append({"role": "user", "content": q})
    st.chat_message("user").write(q)
    resp = w.serving_endpoints.query(
        name=ENDPOINT,
        messages=[ChatMessage(role=ChatMessageRole.USER, content=q)],
    )
    answer = resp.choices[0].message.content
    st.session_state.history.append({"role": "assistant", "content": answer})
    st.chat_message("assistant").write(answer)

# --- app.yaml -----------------------------------------
# command: ["streamlit", "run", "app.py"]
# env:
#   - name: "SERVING_ENDPOINT"
#     value: "unity-airways-agent"

# --- requirements.txt ---------------------------------
# streamlit
# databricks-sdk

# --- deploy -------------------------------------------
# databricks apps create unity-airways-chat
# databricks apps deploy unity-airways-chat --source-code-path "/Workspace/Users/me/unity-airways-chat"
# (or, for CI/CD)  databricks bundle deploy

Field notes

📌 ImportantA deployed app runs as an app service principal — grant that SP access to the endpoint (and any UC data/secrets) it calls; declare them as resources. Keep the agent/LLM logic in the endpoint, not the UI.
💡 Tip · fieldStart from the official e2e-chatbot-app-next template (NextJS + React + AI SDK): streaming, tool-call rendering, persistent history. Use Databricks Asset Bundles (databricks bundle deploy) for repeatable deploys + GitHub-Actions CI/CD.
⚠️ Gotcha · docs-only, evolvingThe project books don't cover Databricks Apps — this is docs-grounded; verify app.yaml keys, CLI flags & the SDK query shape live. Apps run on serverless, billed per hour while running — stop unused apps.
