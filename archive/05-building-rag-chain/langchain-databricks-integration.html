LangChain ↔ Databricks

Two first-party classes — ChatDatabricks and DatabricksVectorSearch — are all you need to wire a Foundation Model and a Vector Search index into a working RAG chain.

01

The big idea

LangChain ships first-party Databricks integrations, so the RAG building blocks work out-of-the-box.

  • ChatDatabricks — wraps a LangChain ChatModel to call any Model Serving endpoint (e.g. a Foundation Model API).
  • DatabricksVectorSearch — wraps a LangChain VectorStore to query a Vector Search index, and .as_retriever() turns it into a retriever.

Both import from one package: from databricks_langchain import ChatDatabricks, DatabricksVectorSearch.

02

Build the chain

Prereqs (book p.140): serverless compute · a pay-per-token Foundation Model API model · a Vector Search endpoint with an index.

retriever → format_docs → prompt → ChatDatabricks → parser
%pip install -U databricks-langchain mlflow
dbutils.library.restartPython()

from databricks_langchain import ChatDatabricks, DatabricksVectorSearch
from langchain_core.prompts import PromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser
import mlflow

mlflow.langchain.autolog()          # trace every chain run (Module 07)

# 1) the model — any Model Serving / Foundation Model endpoint
chat_model = ChatDatabricks(
    endpoint="databricks-claude-3-7-sonnet",
    temperature=0,
    max_tokens=256,
)

# 2) the retriever — a Vector Search index
retriever = DatabricksVectorSearch(
    endpoint="vs_endpoint",
    index_name="catalog.schema.faq_index",
).as_retriever(search_kwargs={"k": 3, "query_type": "ANN"})

# 3) the prompt — merges retrieved context + the user question
prompt = PromptTemplate(
    template=(
        "You are a Unity Airways support assistant. Use ONLY the context.\n\n"
        "Context:\n{context}\n\nQuestion: {question}"
    ),
    input_variables=["context", "question"],
)

def format_docs(docs):
    return "\n\n".join(d.page_content for d in docs)

# 4) compose with LCEL
chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | chat_model
    | StrOutputParser()
)

print(chain.invoke("How do i book flights with Unity Airways?"))
A clean, runnable version of the book's Ch4 chain (which loads the same pieces from a YAML via mlflow.models.ModelConfig).

03

Field notes

📌 ImportantImport from databricks_langchain. ChatDatabricks(endpoint=…) → a served model; DatabricksVectorSearch(index_name=…) → a Vector Search index; .as_retriever() makes it composable with any chain.
💡 Tip · fieldKeep endpoint names, index name & prompt in a YAML config loaded via mlflow.models.ModelConfig — swap models/indexes without touching code. Call mlflow.langchain.autolog() once to trace every run.
⚠️ Gotcha · package + param driftOld tutorials use langchain_community / langchain-databricks — today it's databricks-langchain. Newer docs also show an AI-Gateway form ChatDatabricks(model="…", use_ai_gateway=True); the book/standard form uses endpoint=. Verify for your version.
