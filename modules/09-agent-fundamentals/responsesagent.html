Module 09 — Agent fundamentals and tools · Topic 09.6 · Hands-on ★

ResponsesAgent + Unity Catalog

Wrap the Unity Airways support agent in MLflow's recommended authoring interface, log it as Model-as-Code with its dependent resources, and register it as unity_airways.rag.ua_support_agent. Author on ResponsesAgent — not ChatAgent, not ChatModel.

01 · The big idea

From notebook agent to governed model

Coming out of 09.3 and 09.5 you have a working agent: it retrieves from unity_airways.rag.ua_rag_chunks_index, calls the UC function get_flight_status, and generates with databricks-claude-sonnet-4-5. A notebook object is not a product. Packaging it means three things at once: a reproducible build, a standard streaming tool-aware I/O contract, and a record of the governed services it calls.

  • Author on ResponsesAgent. MLflow's recommended, framework-agnostic interface, built on the OpenAI Responses schema. You get tracing, streaming, multi-turn history, and tool-call history for free. Lineage: ChatModel (legacy) → ChatAgent (superseded) → ResponsesAgent.
  • Implement one method. predict(self, request: ResponsesAgentRequest) -> ResponsesAgentResponse; optionally add predict_stream(...) yielding ResponsesAgentStreamEvents.
  • Log as Model-as-Code (the 05.6 pattern): end agent.py with mlflow.models.set_model(agent), then mlflow.pyfunc.log_model(python_model="agent.py", name="agent", resources=[...]). Store the code, not a pickle.
  • Declare resources + register to UC. List the LLM endpoint, the index, and the UC function so deployment mints scoped credentials; register as unity_airways.rag.ua_support_agent. Then deploy via agents.deploy(...) (Module 11) or Databricks Apps (Module 10.5).

03 · How it works

Wrap the agent, implement predict

A ResponsesAgent subclass stores your framework agent and exposes it through the standard Responses contract. predict is the one method you must implement; predict_stream is optional but recommended, and a clean predict just drives it and keeps the completed items.

from typing import Generator
from mlflow.pyfunc import ResponsesAgent
from mlflow.types.responses import (
    ResponsesAgentRequest, ResponsesAgentResponse, ResponsesAgentStreamEvent,
)

class UASupportAgent(ResponsesAgent):
    def __init__(self, agent):
        self.agent = agent   # the LangGraph / LangChain agent from 09.5

    def predict(self, request: ResponsesAgentRequest) -> ResponsesAgentResponse:
        outputs = [
            event.item
            for event in self.predict_stream(request)
            if event.type == "response.output_item.done"
        ]
        return ResponsesAgentResponse(output=outputs, custom_outputs=request.custom_inputs)

    def predict_stream(
        self, request: ResponsesAgentRequest
    ) -> Generator[ResponsesAgentStreamEvent, None, None]:
        for chunk in self._run_agent_streaming(request.input):
            yield ResponsesAgentStreamEvent(
                **self.create_text_delta(delta=chunk.text, item_id=chunk.id)
            )
        yield ResponsesAgentStreamEvent(type="response.output_item.done", item=final_item)
  • Framework-agnostic. The wrapper adapts any underlying agent (LangGraph, LangChain, raw Python) to the OpenAI Responses schema, so existing OpenAI-compatible chat UIs connect with no rewrite.
  • Only predict / predict_stream are yours. create_text_delta(delta=..., item_id=...) is a base-class helper for token events. The framework↔Responses converters (📘B1 calls them _responses_to_cc() / _langchain_to_responses()) are functions you write — do not invent other methods on the base class.
  • Tracing for free. mlflow.langchain.autolog() captures every step, input, output, and tool call as an MLflow trace when the agent runs.
⚠️ GotchaThe two books predate parts of this surface. 📘B1 Ch7 does use ResponsesAgent, but older blogs and 📗B2 still show ChatModel / ChatAgent. When a customer's code subclasses those, that is the legacy path — the migration target is ResponsesAgent. The SplitChatMessagesRequest / StringResponse helpers are deprecated.
04 · Run this

Wire, log, verify, register

Prereqs: serverless or DBR ML runtime, MLflow ≥ 3.1; the 09.3 tools in UC (unity_airways.rag.ua_rag_chunks_index, unity_airways.rag.get_flight_status); access to databricks-claude-sonnet-4-5; and rights to create a model in unity_airways.rag (USE CATALOG + USE SCHEMA + CREATE MODEL).

%pip install -U mlflow databricks-langchain langgraph
dbutils.library.restartPython()

1. In agent.py — wire the tools and LLM from databricks-langchain:

from databricks_langchain import ChatDatabricks, VectorSearchRetrieverTool, UCFunctionToolkit

retriever_tool = VectorSearchRetrieverTool(
    index_name="unity_airways.rag.ua_rag_chunks_index",
    num_results=5,
    tool_description="Search Unity Airways policies and FAQ: cancellations, baggage, refunds.",
)
uc_toolkit = UCFunctionToolkit(function_names=["unity_airways.rag.get_flight_status"])

llm   = ChatDatabricks(endpoint="databricks-claude-sonnet-4-5")
tools = [retriever_tool, *uc_toolkit.tools]
# agent = your LangGraph/LangChain reasoning loop over (llm, tools) from 09.5

2. Still in agent.py — wrap in ResponsesAgent and set the model (last line):

import mlflow
mlflow.langchain.autolog()
responses_agent = UASupportAgent(agent)
mlflow.models.set_model(responses_agent)   # <-- this object is the model

3. Smoke-test in the notebook, then log as Model-as-Code with resources:

from mlflow.models.resources import (
    DatabricksServingEndpoint, DatabricksVectorSearchIndex, DatabricksFunction,
)

with mlflow.start_run():
    logged_agent = mlflow.pyfunc.log_model(
        python_model="agent.py",          # log the code, not the object
        name="agent",
        resources=[
            DatabricksServingEndpoint(endpoint_name="databricks-claude-sonnet-4-5"),
            DatabricksVectorSearchIndex(index_name="unity_airways.rag.ua_rag_chunks_index"),
            DatabricksFunction(function_name="unity_airways.rag.get_flight_status"),
        ],
        pip_requirements=["mlflow", "databricks-langchain", "langgraph"],
    )

4. Pre-deployment check — load + predict in a clean env:

mlflow.models.predict(
    model_uri=f"runs:/{logged_agent.run_id}/agent",
    input_data={"input": [{"role": "user", "content": "Is flight UA118 on time?"}]},
    env_manager="uv",
)

5. Register to Unity Catalog (creates version 1):

mlflow.set_registry_uri("databricks-uc")
UC_MODEL = "unity_airways.rag.ua_support_agent"   # catalog.schema.model
registered = mlflow.register_model(model_uri=logged_agent.model_uri, name=UC_MODEL)
print(registered.name, registered.version)        # unity_airways.rag.ua_support_agent 1

6. Hand off to deployment (forward reference — not taught here):

# Module 11 — Agent Framework deploy: serving endpoint + review app + feedback model
from databricks import agents
agents.deploy(UC_MODEL, registered.version)
# Or package it behind a Databricks App — Module 10.5.
05 · Field notes

What to remember

📌 ImportantThe topic reduces to five moves: wrap it (ResponsesAgent subclass), declare it (set_model in agent.py), log it (pyfunc.log_model with resources), verify it (mlflow.models.predict), register it (catalog.schema.model in UC). Deployment then reads the registered version. Author on ResponsesAgent; recognize ChatAgent/ChatModel only as the migration path.
💡 Tip · fieldKeep agent.py self-contained and importable — no notebook-only globals, no dbutils at import time. That is what lets MLflow re-execute it cleanly at load and makes the mlflow.models.predict check trustworthy. Put endpoint / index / function names at the top of the file so the resource list and the agent read the same constants. Implement predict_stream even if your first consumer doesn't stream — then predict is a three-line wrapper over it.
⚠️ Gotcha📘B1 Ch7 registers this agent as workspace.unity_airways.unity-airways-booking-agent (catalog workspace, schema unity_airways, hyphenated name). This project standardizes on CATALOG="unity_airways", SCHEMA="rag", model ua_support_agent. Use the project's three-level name; avoid hyphenated model names. And never omit a resource — a deployed agent that calls an unlisted endpoint, index, or function fails auth at run time.
