Module 07 — MLflow Tracing and observability · Topics 07.2-07.3 · Theory + Hands-on ★

Tracing a GenAI App

Turn one confident-but-wrong answer into a request you can read. autolog() traces the chain for free; @mlflow.trace and start_span add your own steps — and it all lands in one trace tree. Automated plus manual, on the Unity Airways RAG chain.

01 · The big idea

From black box to a request tree

A customer asks "Can I get a refund on a Basic Economy fare?" and the Unity Airways bot answers confidently and wrong. All you have is the input and the output — but in between sits a compound system: retriever, prompt, model, parser. Tracing captures that whole journey per request so you can see which step failed instead of guessing.

  • Automated tracing is one line: mlflow.langchain.autolog(). Every chain.invoke(...) then emits a Trace with nested Spans — retriever (chunks + scores), prompt (rendered messages), LLM (answer + tokens + latency). No per-call code.
  • Sibling autologgers cover the other SDKs: mlflow.openai.autolog(), mlflow.anthropic.autolog(), plus LangGraph, LlamaIndex, and 20+ more.
  • Manual tracing adds spans autolog can't see — your own pre/post steps. Two fluent forms: the @mlflow.trace decorator on a function, and with mlflow.start_span(...) as span: for a block, where you set inputs, outputs, and attributes yourself.
  • Auto and manual combine into ONE trace. MLflow links parent/child by call nesting, so a manual pre-step, the autolog'd chain, and a manual rerank all sit under a single request tree. mlflow.set_active_model(...) ties those traces to the LoggedModel from Module 06 — the thread Module 08 evaluation pulls on.

03 · How it works

Automated first, then layer manual on top

The decision is per step, not per app. If a step runs inside a library MLflow can autolog, you get its span for free. If it is your own code, you add a span by hand. Both forms produce spans that land in the same trace.

Autolog covers the framework calls; manual covers the glue you wrote. MLflow merges them into one trace.
  • Automated — one line, whole chain. mlflow.langchain.autolog() hooks LangChain's callbacks; every runnable .invoke() emits a span. Best practice (Book 1): start here, then add manual spans for the gaps.
  • Manual form 1 — the decorator. @mlflow.trace(name=, span_type=, attributes=) wraps a whole function; its args become span inputs and its return becomes span outputs, automatically. Keep it the outermost decorator.
  • Manual form 2 — the context manager. with mlflow.start_span(name=..., span_type=...) as span: traces an arbitrary block; you call set_inputs, set_outputs, set_attributes yourself. Use it when there is no single function to decorate.
  • They combine automatically. Wrap steps in an outer @mlflow.trace orchestrator; call your manual helpers and the autolog'd pieces inside. MLflow detects the nesting — no code links parent to child. Fanning out with ThreadPoolExecutor? Copy the context: contextvars.copy_context().run(fn, ...).
  • RETRIEVER is the one special type. Output mlflow.entities.Document (page_content, metadata, id) so docs render as documents and retrieval can be evaluated later.
  • Bind traces to a version. mlflow.set_active_model(name="ua_rag_chain") stamps traces with the Module 06 LoggedModel, so Module 08 evaluation reads exactly this build's traces.
04 · Run this

autolog, then manual, then combine

Prereqs: serverless or DBR ML runtime, MLflow ≥ 3.1; the Module 05 chain (and its retriever, prompt, llm, format_docs) and the Module 06 model unity_airways.rag.ua_rag_chain.

%pip install -U "mlflow[databricks]>=3.1" databricks-langchain databricks-vectorsearch
dbutils.library.restartPython()

0. Imports and variables:

import mlflow
from mlflow.entities import SpanType, Document

CATALOG  = "unity_airways"
SCHEMA   = "rag"
UC_MODEL = f"{CATALOG}.{SCHEMA}.ua_rag_chain"
QUESTION = "Can I get a refund on a Basic Economy fare?"

1. Turn on automated tracing and pin the version:

mlflow.langchain.autolog()                     # every LangChain .invoke() now traces
active = mlflow.set_active_model(name="ua_rag_chain")   # link traces to the LoggedModel
print(active.name, active.model_id)

2. Read the automated trace — open the cell's inline MLflow Trace UI and inspect the RETRIEVER span:

answer = chain.invoke(QUESTION)
print(answer)
# Open the RETRIEVER span: did the refund/fare-rules chunk come back?
# If not, the bug is upstream in chunking/retrieval, not the model.

3. Manual pre-step with the decorator:

@mlflow.trace                                   # default span type; name = function name
def preprocess_question(raw: str) -> str:
    return raw.strip().replace("BE fare", "Basic Economy fare")

4. Manual mid-step with the context manager:

def rerank(question, docs):
    with mlflow.start_span(name="rerank", span_type=SpanType.RERANKER) as span:
        span.set_inputs({"question": question, "n_candidates": len(docs)})
        ranked = sorted(docs, key=lambda d: len(d.page_content))[:3]   # stand-in scorer
        span.set_outputs({"kept": [d.metadata.get("source_doc") for d in ranked]})
        span.set_attributes({"reranker": "heuristic", "top_k": 3})
        return ranked

5. Orchestrate — one function, one trace, auto + manual interleaved:

from langchain_core.output_parsers import StrOutputParser

@mlflow.trace(span_type=SpanType.CHAIN)         # outer span = the trace root
def answer_unity_airways(raw_question: str) -> str:
    q = preprocess_question(raw_question)       # manual span
    docs = retriever.invoke(q)                  # AUTO RETRIEVER span
    docs = rerank(q, docs)                      # manual RERANKER span
    return (prompt | llm | StrOutputParser()).invoke(   # AUTO prompt + LLM spans
        {"context": format_docs(docs), "question": q}
    )

print(answer_unity_airways("Can I get a refund on a BE fare?"))

Verify: the newest trace reads answer_unity_airways (CHAIN) → preprocess_question → retriever (RETRIEVER) → rerank (RERANKER) → prompt → ChatDatabricks (CHAT_MODEL). Auto and manual spans in one tree.

6. Trace-level tags for later search:

@mlflow.trace(span_type=SpanType.CHAIN)
def answer_tagged(raw_question: str) -> str:
    mlflow.update_current_trace(tags={"app": "ua-policy-bot", "channel": "web"})
    return answer_unity_airways(raw_question)
# Filter by these in Experiments -> your experiment -> Traces (not on the span view).

7. Make a RETRIEVER span render as documents (needed for evaluation later):

@mlflow.trace(span_type=SpanType.RETRIEVER, name="ua_retriever")
def retrieve_as_documents(q: str):
    raw = retriever.invoke(q)                    # LangChain Documents
    return [Document.from_langchain_document(d) for d in raw]   # MLflow Documents
05 · Field notes

What to remember

📌 ImportantThe whole lesson is two moves: turn on autolog() to get the framework spans for free, then add manual spans (@mlflow.trace for a function, mlflow.start_span for a block) for the code you wrote. MLflow merges both into one trace automatically — you never wire parent to child by hand. Trace = TraceInfo + TraceData, and TraceData is the list of spans.
💡 Tip · fieldMake reading the trace your first debugging step, not your last. When a RAG answer is wrong, open the RETRIEVER span before you touch the prompt or swap the model — nine times out of ten the right chunk was never retrieved, and the trace tells you that in one click.
⚠️ GotchaOn serverless compute, GenAI autologging is not auto-enabled — call mlflow.<library>.autolog() explicitly for each integration. Book 1 is an O'Reilly Early Release and predates parts of MLflow 3, but its tracing APIs (autolog(), @mlflow.trace, start_span, SpanType, set_inputs/outputs/attributes, update_current_trace, Document) are current and were re-verified live. The low-level MlflowClient span API and the exact set_active_model-to-LoggedModel linking are the two spots to re-confirm for your runtime.
