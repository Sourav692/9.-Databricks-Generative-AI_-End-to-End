01

The big idea

You can't build GenAI on Databricks until the platform plumbing is in place: a Lakehouse workspace, compute, Unity Catalog governance, and a way to find data and models.

  • Lakehouse — one platform storing raw + structured data as Delta, governed by Unity Catalog, run on shared compute (serverless or classic).
  • Unity Catalog — the governance spine: a single three-level namespace catalog.schema.object for tables, volumes (files), functions, models, and vector indexes.
  • Mosaic AI — the umbrella for building, serving, and governing AI; a stack of modular pieces, not one product. Cornerstone deep-dive: mosaic-ai-landscape.html.
  • Unity Airways — the running airline-assistant use case; Databricks Marketplace — how you pull in ready-made data and models (e.g. NOAA weather) via Delta Sharing.

03

Foundation in practice

00.3 [Hands-on] — confirm the workspace can run code (book Ch1, Step 13):

# Attach the notebook to Serverless (or a classic ML cluster), then run:
print("Databricks environment is ready")

00.2 [Theory] — the three-level namespace and a governance grant. Everything lives at catalog.schema.object, including volumes for files:

-- structured records: catalog.schema.table
SELECT * FROM main.airways.bookings;

-- non-tabular files (RAG PDFs) governed as a volume
-- read them at /Volumes/main/airways/faq_docs/...
GRANT READ VOLUME ON VOLUME main.airways.faq_docs
  TO `data-scientists`;

00.6 [Hands-on] — after Get instant access in Marketplace, the dataset appears as a read-only catalog; query it with the three-part namespace:

-- a Marketplace listing (e.g. NOAA / weather) installs as a read-only catalog
SELECT * FROM weather_data.noaa.daily_summaries LIMIT 10;

Catalog, schema, table, and volume names above are illustrative — create your own and confirm the exact listing name in the live Marketplace.

04

Field notes

📌 ImportantUnity Catalog's three-level namespace catalog.schema.object is the single most important idea in Module 00 — tables, volumes, functions, models, and vector indexes all live there and inherit its permissions, lineage, and audit.
💡 Tip · fieldDefault to serverless compute (no cluster tuning) and put every RAG document in a UC volume. That removes two of the most common beginner blockers in one move.
⚠️ GotchaThe book's Ch1 setup is Azure-portal-specific and uses a classic ML cluster; current Databricks leads with serverless and cross-cloud sign-up. Databricks is also moving away from "Mosaic AI" branding (Vector Search is surfaced as AI Search). Verify the live product name before quoting it.
