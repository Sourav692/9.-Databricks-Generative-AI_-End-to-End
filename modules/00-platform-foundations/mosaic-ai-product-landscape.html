01

The big idea

"Mosaic AI" is not one product — it is a stack of modular pieces that snap together. Learn the layers first, then the boxes inside them.

Two pieces run across every layer and are easy to forget: Unity Catalog (governance) and MLflow + the AI Gateway (quality & ops). Hold the layered map in your head and every later entry in this almanac becomes a deep-dive into a single box.

03

The cert book's 3-layer view

For the exam, compress the infrastructure into three core layers (📗 B2 Ch9):

LayerRole
Model serving layerHosts foundation, embedding & fine-tuned models behind scalable API endpoints.
Feature & data integration layerFeeds models current context from Delta tables, Vector Search indexes, or streaming pipelines.
Orchestration layerScales resources, balances workloads, routes requests, caches responses, meets SLOs.
04

Field notes

📌 ImportantMosaic AI is a stack, not a product. Unity Catalog + AI Gateway + MLflow/Tracing are cross-cutting. There are two ways to build agents: Agent Framework (code) vs Agent Bricks (no/low-code).
💡 Tip · fieldMinimum "chatbot over our docs" stack = 4 boxes: Vector Search → Foundation Model API → Agent Framework/Bricks → Model Serving, governed by Unity Catalog. Enable dynamic batching for many short requests.
⚠️ Gotcha · names drift"Databricks Vector Search" → also "AI Search"; "MLflow AI Gateway" → "Unity/Databricks AI Gateway". Confirm current naming in the docs before quoting a customer.

Notes & self-check
