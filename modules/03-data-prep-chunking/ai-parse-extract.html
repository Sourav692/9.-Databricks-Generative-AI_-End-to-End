Document Parsing and Extraction with AI Functions

Turn messy PDFs and scans in a Unity Catalog volume into clean, chunkable text and structured fields with two SQL functions — no OCR stack, no prompt wrangling, everything stays in Delta.

01

The big idea

Two built-in SQL functions do the document data-prep that used to need a Python OCR project. ai_parse_document reads bytes into text and layout; ai_extract pulls named fields out of that text.

  • Pipeline: volume PDF → ai_parse_document → text and layout → ai_extract → Delta table ready to chunk.
  • Task-specific first: use these before the general ai_query; escape to ai_query only for deeply nested JSON or image reasoning.
  • Governed and in-platform: source bytes in a UC volume, output in a UC Delta table, no data leaving to an external OCR API.
  • Both GA, billed under the AI_FUNCTIONS product, and run on serverless compute only.
  • Requirements churn — treat runtime/region notes here as a dated snapshot and re-verify before a demo.

03

ai_parse_document — bytes to structured text

Signature: ai_parse_document(content [, options]). content is BINARY; options is a MAP<STRING, STRING> (version, imageOutputPath, descriptionElementTypes, pageRange). It returns a VARIANT describing the document.

{
  "document": {
    "pages":    [ { "id": INT, "image_uri": STRING } ],
    "elements": [ { "id": INT, "type": STRING, "content": STRING,
                    "confidence": DOUBLE,
                    "bbox": [ { "coord": [INT], "page_id": INT } ],
                    "description": STRING } ]
  },
  "error_status": [ { "error_message": STRING, "page_id": INT } ],
  "metadata": { "id": STRING, "version": STRING }
}
  • Element type: text, table, figure, title, caption, section_header, page_header, page_footer, page_number, footnote. Elements come back in reading order, which is what fixes scrambled multi-column pages.
  • Formats: PDF, JPG/JPEG, PNG, TIFF/TIF, DOC/DOCX, PPT/PPTX.
  • Limits: 100 MB max file size; 500 pages max (over 500 fails immediately).
⚠️ Gotcha · error pathThe top-level error field is error_status — an array of per-page {error_message, page_id} objects (empty/absent on a clean parse), NOT a scalar error. Because it is an array, test the first element: keep clean rows with WHERE parsed:error_status[0] IS NULL and send failures (WHERE parsed:error_status[0] IS NOT NULL) to a diagnostics/quarantine table.
04

ai_extract — text to named fields

Two versions, and the difference changes how you read the output:

  • v2.1 (current): ai_extract(content, schema [, options]). schema is a JSON string — a simple array of names, or a typed object.
  • v1 (legacy): ai_extract(content, labels [, options]), where labels is an ARRAY<STRING>; returns flat string fields.

content is text or the ai_parse_document VARIANT. Options: version, instructions, enableCitations, enableConfidenceScores. The v2.1 output wraps each field as an object:

{
  "response": {
    "effective_date": { "value": "2026-01-01",
                        "citation_ids": [ ... ],
                        "confidence_score": 0.98 }
  },
  "error_message": null,
  "metadata": { ... }
}

Schema limits: types string / integer / number / boolean / enum; up to 500 enum values; 128 fields; 7 nesting levels; 150-char field names; 128,000-token input.

⚠️ Gotcha · value pathBecause v2.1 wraps each field in an object, the value path is result:response:effective_date:value::STRING — not a bare scalar. Inspect the VARIANT once and confirm the path before hardcoding it.
05

Do it on DatabricksHands-on

Step 1 — parse every document in the volume:

CREATE OR REPLACE TABLE unity_airways.rag.policy_parsed AS
SELECT path, ai_parse_document(content) AS parsed   -- BINARY in, VARIANT out
FROM read_files('/Volumes/unity_airways/rag/policy_docs/', format => 'binaryFile');

Verify: error_status is empty (its first element is null) and the first element has real text.

SELECT path,
       parsed:error_status                          AS errors,
       parsed:document:elements[0]:content::STRING  AS first_text_block
FROM unity_airways.rag.policy_parsed LIMIT 10;

Step 2 — flatten elements to text (cast to ARRAY<VARIANT> before you touch it):

CREATE OR REPLACE TABLE unity_airways.rag.policy_text AS
SELECT path,
       concat_ws('\n',
         transform(
           variant_get(parsed, '$.document.elements', 'ARRAY<VARIANT>'),
           e -> variant_get(e, '$.content', 'STRING'))) AS doc_text
FROM unity_airways.rag.policy_parsed
WHERE parsed:error_status[0] IS NULL;

Step 3 — extract structured fields with a typed schema:

CREATE OR REPLACE TABLE unity_airways.rag.policy_extracted AS
SELECT path, doc_text,
  ai_extract(
    doc_text,
    '{"policy_title":       {"type":"string"},
      "effective_date":     {"type":"string","description":"yyyy-mm-dd"},
      "refund_window_days": {"type":"integer"},
      "fare_class":         {"type":"enum","labels":["Economy","Premium","Business","First"]}}',
    map('instructions','These are Unity Airways passenger policy documents.')
  ) AS fields
FROM unity_airways.rag.policy_text;

Step 4 — land a clean Delta table (confirm the value path first):

CREATE OR REPLACE TABLE unity_airways.rag.policy_structured AS
SELECT path, doc_text,
       fields:response:policy_title:value::STRING     AS policy_title,
       fields:response:effective_date:value::STRING   AS effective_date,
       fields:response:refund_window_days:value::INT  AS refund_window_days,
       fields:response:fare_class:value::STRING       AS fare_class,
       fields:error_message::STRING                   AS extract_error
FROM unity_airways.rag.policy_extracted;

How to verify the whole thing worked: counts line up and fields look sane — count(policy_title) should track count(*), and count_if(extract_error IS NOT NULL) should be near zero.

06

Worked example — Unity Airways

  • Parse: 1,200 files — native PDFs, DOCX guides, ~300 scans. One ai_parse_document call covers all formats; the scans that pypdf returned empty for now come back with full OCR text.
  • Errors: a 640-page merged archive (over the 500-page cap) and a digitally signed PDF land in error_status and go to a review table.
  • Flatten: reading order fixes the two-column contract of carriage; dropping page_number and page_footer elements cuts chunk noise.
  • Extract + land: the enum on fare_class keeps cabins clean; retrieval can later filter by fare_class and effective_date.
📌 ImportantKeep the parsed VARIANT table even after flattening to text. It is your audit trail (bounding boxes, element types) and lets you re-derive text without paying to re-parse.

Field notes

📌 ImportantPrefer the task-specific functions (ai_parse_document, ai_extract, ai_classify) for document prep. ai_query is the fallback for what they cannot do, not the default.
💡 Tip · fieldThe same functions run from PySpark via expr("ai_parse_document(content)") inside withColumn — pick SQL or PySpark to match the surrounding job.
⚠️ Gotcha · docs vs booksCompute moved to serverless-only and the overview now lists DBR 18.2+, while the ai_parse_document page lists 17.3+. The study guide predates these functions. Teach the current requirement and re-verify — minimums have been rising, and both functions are region-limited.
