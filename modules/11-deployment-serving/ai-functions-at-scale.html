AI Functions at scale

Run a Foundation Model over a whole Delta table in SQL. Delta in, AI Function in the SELECT, Delta out, scheduled by a job.

01 · The big idea

AI as a column

AI Functions are built-in SQL functions that call Foundation Models straight from a query — no endpoint to create, no API key, no client code.

  • ai_query is the general workhorse: point it at any served model (a Foundation Model or your own agent endpoint) and run a prompt over every row. Ask for structured JSON with responseFormat or returnType.
  • Task-specific functions (ai_classify, ai_analyze_sentiment, ai_extract, ai_mask, ai_summarize, ai_translate, ai_fix_grammar, ai_gen) are pre-tuned one-liners. Prefer them when the task fits.
  • vector_search queries a Databricks AI Search index from SQL, so retrieval joins your data.
  • This is batch inference, not a chatbot. Use it when nobody is waiting on the answer.
📌 The one patternDelta table in → AI Function in the SELECT → Delta table out → Lakeflow Job on a schedule. Every example below is a variation of this.

03 · Hands-on

Runnable SQL

Running example · Unity Airways

One Delta table of raw feedback. Every query below enriches it and writes structured columns back.

CREATE TABLE IF NOT EXISTS unity_airways.rag.ua_support_tickets (
  ticket_id   BIGINT,
  received_at TIMESTAMP,
  channel     STRING,   -- 'email' | 'chat' | 'app_review' | 'call_transcript'
  language    STRING,   -- ISO code, e.g. 'en','es','fr'
  raw_text    STRING    -- customer message / review body (may contain PII)
);
Lab A — enrich the whole table with task functions 
CREATE OR REPLACE TABLE unity_airways.rag.ua_tickets_enriched AS
SELECT
  ticket_id, received_at, channel,
  ai_classify(raw_text,
    '{"baggage":"lost, delayed, or damaged luggage",
      "delay":"flight delays, cancellations, missed connections",
      "billing":"refunds, charges, fare disputes",
      "other":"anything else"}')                       AS intent,
  ai_analyze_sentiment(raw_text)                        AS sentiment,
  ai_summarize(raw_text, 20)                            AS one_liner,
  ai_mask(raw_text, ARRAY('person','email','phone','address')) AS safe_text
FROM unity_airways.rag.ua_support_tickets
WHERE raw_text IS NOT NULL;   -- NULL in = NULL out; filter to save calls
💡 VerifySELECT intent, sentiment, count(*) FROM ...ua_tickets_enriched GROUP BY 1,2 ORDER BY 3 DESC; — sensible buckets, and no raw names or emails left in safe_text.
Lab B — ai_query with structured JSON output 
CREATE OR REPLACE TABLE unity_airways.rag.ua_tickets_structured AS
SELECT
  ticket_id,
  ai_query(
    'databricks-claude-sonnet-4-5',
    CONCAT('Extract as JSON with keys intent, urgency (1-5), ',
           'refund_requested (bool), affected_flight. Ticket: ', raw_text),
    responseFormat  => '{"type":"json_object"}',
    failOnError     => false,                  -- batch safety
    modelParameters => named_struct('temperature', CAST(0.0 AS DOUBLE), 'max_tokens', 300)
  ) AS ai_response
FROM unity_airways.rag.ua_support_tickets
WHERE raw_text IS NOT NULL;

-- parse JSON into columns; split failures off with ai_response.errorMessage
SELECT ticket_id,
  from_json(ai_response.response,
    'STRUCT<intent:STRING, urgency:INT, refund_requested:BOOLEAN, affected_flight:STRING>') AS fields,
  ai_response.errorMessage AS error
FROM unity_airways.rag.ua_tickets_structured;
Lab C — call your own agent endpoint + vector_search 
-- ai_query against a CUSTOM endpoint (Module 09 ResponsesAgent)
SELECT ticket_id,
       ai_query('ua-support-agent',
                CONCAT('Draft a reply to: ', raw_text),
                failOnError => false).response AS draft_reply
FROM unity_airways.rag.ua_support_tickets
WHERE raw_text IS NOT NULL;

-- retrieve top policy chunks from an AI Search index, inside SQL
SELECT t.ticket_id,
       vector_search(
         index       => 'unity_airways.rag.ua_rag_chunks_index',
         query_text  => t.raw_text,
         num_results => 3
       ) AS top_policy_chunks
FROM unity_airways.rag.ua_support_tickets t
WHERE t.raw_text IS NOT NULL;
Lab D — translate, fix grammar, generate; and the PySpark form 
SELECT
  ai_translate(raw_text, 'en')   AS text_en,     -- supported: en,de,fr,it,pt,hi,es,th
  ai_fix_grammar(raw_text)       AS text_clean,
  ai_gen(CONCAT('Write a one-line apology for: ', raw_text)) AS apology_line
FROM unity_airways.rag.ua_support_tickets
WHERE raw_text IS NOT NULL AND channel = 'app_review';
from pyspark.sql.functions import expr

(spark.table("unity_airways.rag.ua_support_tickets")
      .filter("raw_text IS NOT NULL")
      .withColumn("sentiment", expr("ai_analyze_sentiment(raw_text)"))
      .withColumn("intent",    expr("ai_classify(raw_text, array('baggage','delay','billing','other'))"))
      .write.mode("overwrite").saveAsTable("unity_airways.rag.ua_tickets_enriched"))

04 · Decision

Batch AI Function vs real-time endpoint

The first question is always: is a human waiting on this answer? If yes, it belongs on a real-time Model Serving endpoint (11.1). If it is scheduled or bulk, it belongs in an AI Function.

ai_query has no conversational state, tool-calling, or multi-step logic. Source: B2 Ch9, Table 9-12.
Suitable for ai_queryUse Model Serving (11.1)
TimingScheduled / recurring / batchUser-driven, real-time
Data sizeLarge tables, bulkSmall, individual requests
LatencySeconds to minutes OKMilliseconds
ExamplesEnrichment, embeddings, evalChatbots, live RAG, agents
Runs onSQL warehouse / DBSQL jobModel Serving endpoint
05 · Field notes

Important, tips, gotchas

📌 Importantai_query is batch inference on tables, not a chat API. No conversational state, tool-calling, or multi-step logic. Match the workload to the engine: bulk/scheduled → AI Functions; a human waiting → Model Serving (11.1).
💡 Tip · fieldDevelop on a sample (... WHERE raw_text IS NOT NULL LIMIT 200), confirm output shape and cost, then remove the LIMIT. Wrap a common ai_query prompt in a SQL UDF so analysts call classify_ticket(text) and you swap the model in one place.
⚠️ Gotcha · model driftOlder docs show databricks-claude-sonnet-4; a current valid snapshot is databricks-claude-sonnet-4-5 (newer Claude Sonnet/Opus revisions may also be listed). Endpoint names churn — verify on the supported-models page and never guess. Always set failOnError => false in a batch job so one bad row does not fail the whole run.
⚠️ Gotcha · runtime gatesresponseFormat needs DBR 15.4+, returnType 15.2+, failOnError/modelParameters 15.3+, ai_parse_document 17.3+. AI Functions want a Pro or Serverless SQL warehouse (not Classic).
