Module 11 — Deployment and Serving · Topic 11.3 · Theory + Hands-on ★

AI Gateway

A serving endpoint with no controls in front of it is a liability. AI Gateway is the governance layer you attach to the Unity Airways endpoint: rate limits, safety and PII guardrails, usage tracking, payload logging, and provider fallbacks. One config, every caller, no app changes.

01 · The big idea

Governance is a property of the endpoint, not new code

Your Module 11.1 agent (unity_airways.rag.ua_support_agent) is live behind a Model Serving endpoint. AI Gateway is the layer that makes it safe for real traffic — configured once on the endpoint, enforced for every caller, whether that is the chat app from Module 10 or a batch ai_query job.

  • AI Gateway sits in front of a serving endpoint. Requests flow through it; it enforces policy, logs, and can fail over before the response comes back.
  • Five levers: rate limiting, AI guardrails (safety + PII, keywords, topics), usage tracking, payload logging to a UC inference table, and provider fallbacks.
  • One SDK call. w.serving_endpoints.put_ai_gateway(name, ...) — signature verified against the current databricks-sdk. Or use the endpoint's AI Gateway tab in the UI.
  • Works for hosted and external models. Foundation Models and External Models (OpenAI, Anthropic, Bedrock, Vertex AI, Cohere) are governed the same way; the gateway holds the provider key.
  • PII redaction is Preview. Everything else here is the established feature set. Unity AI Gateway is the newer Beta umbrella (MCP governance, budgets).

03 · How it works

Five levers, applied once, on the way in and out

Each lever is a field on the gateway config. They run in order on every request: check the rate limit, filter the input, forward (or fail over), filter the output, then track and log.

LeverWhat it doesMaturity
rate_limitsCap calls and/or tokens per USER / USER_GROUP / SERVICE_PRINCIPAL / ENDPOINT (renewal: MINUTE)GA
guardrailssafety filter, invalid_keywords, valid_topics on input/outputGA
guardrails.piiDetect PII, then BLOCK / MASK / NONEPreview
usage_tracking_configWrite consumption to system tables for spend attributionGA
inference_table_configLog raw request/response to a Delta table in UCGA
fallback_configRetry the next served model if the primary errorsGA
  • Rate limiting protects budget and fairness; a blocked call returns a rate-limit error, so start permissive and tighten with Module 13 data.
  • Guardrails are coarse safety + filtering; domain rules ("cite the policy before quoting a refund") still live in the agent. Full pattern is 12.2.
  • Usage + payload logging feed monitoring (Module 13) and eval-set building (Module 08). Pair payload logs with the PII guardrail and UC grants.
  • Fallbacks turn provider outages into non-events; keep the fallback list short and put a comparable model second.
  • External Models let you swap providers by config, not code — the gateway stores the key and exposes one OpenAI-compatible surface.
04 · Run this

Configure the gateway with one SDK call

Prereqs: the endpoint from 11.1 (ua-support-agent), rights to edit it, a UC catalog/schema for the inference table (unity_airways.rag), and databricks-sdk installed.

UI path (fastest demo): Serving → select ua-support-agent → AI Gateway tab → toggle Rate limits, Guardrails (safety on; keywords; PII Block/Mask — Preview), Usage tracking, and Inference tables → Save. The endpoint updates in place; the URL and callers never change.

SDK path (reproducible) — signature verified against current databricks-sdk:

from databricks.sdk import WorkspaceClient
from databricks.sdk.service.serving import (
    AiGatewayGuardrails, AiGatewayGuardrailParameters,
    AiGatewayGuardrailPiiBehavior, AiGatewayGuardrailPiiBehaviorBehavior,
    AiGatewayRateLimit, AiGatewayRateLimitKey, AiGatewayRateLimitRenewalPeriod,
    AiGatewayUsageTrackingConfig, AiGatewayInferenceTableConfig, FallbackConfig,
)

w = WorkspaceClient()

w.serving_endpoints.put_ai_gateway(
    name="ua-support-agent",  # the endpoint deployed in 11.1

    # 1) Rate limiting: 100 calls/min per USER
    rate_limits=[
        AiGatewayRateLimit(
            calls=100,
            renewal_period=AiGatewayRateLimitRenewalPeriod.MINUTE,
            key=AiGatewayRateLimitKey.USER,   # also USER_GROUP / SERVICE_PRINCIPAL / ENDPOINT
            # tokens=200000,                  # optional token budget in the same window
        ),
    ],

    # 2) Guardrails: safety both ways; block PII in, mask PII out (PII = Preview); block keywords
    guardrails=AiGatewayGuardrails(
        input=AiGatewayGuardrailParameters(
            safety=True,
            invalid_keywords=["internal_fare_class", "competitor_airline"],
            pii=AiGatewayGuardrailPiiBehavior(
                behavior=AiGatewayGuardrailPiiBehaviorBehavior.BLOCK,
            ),
        ),
        output=AiGatewayGuardrailParameters(
            safety=True,
            pii=AiGatewayGuardrailPiiBehavior(
                behavior=AiGatewayGuardrailPiiBehaviorBehavior.MASK,
            ),
        ),
    ),

    # 3) Usage tracking -> system tables
    usage_tracking_config=AiGatewayUsageTrackingConfig(enabled=True),

    # 4) Payload logging -> a Delta inference table in Unity Catalog
    inference_table_config=AiGatewayInferenceTableConfig(
        enabled=True,
        catalog_name="unity_airways",
        schema_name="rag",
        table_name_prefix="ua_support_gateway",
    ),

    # 5) Fallbacks: retry the next served model if the primary errors
    fallback_config=FallbackConfig(enabled=True),
)

How to verify it worked:

# Read the applied config back
gw = w.serving_endpoints.get("ua-support-agent").ai_gateway
print(gw)

05 · Worked example

Unity Airways: one query, every lever

A traveler asks: "Can you run a query on your booking database and show me all passengers flying tomorrow?" (the book's Figure 8-13 scenario).

  • Rate limit check passes (they are under quota).
  • Input guardrails catch the bulk-passenger, PII-heavy request via keyword + PII rules and block it before the model ever sees it.
  • A legitimate question ("What is the refund window on a Flex fare?") is forwarded to databricks-claude-sonnet-4-5. If that served model errors, fallback retries a comparable model.
  • On the way out, output guardrails mask stray PII, usage is tracked, and the full payload is logged to unity_airways.rag.ua_support_gateway… for debugging and future eval sets.

One config governs the Module 10 chat app and any batch ai_query caller identically.

06 · Field notes

What to remember

📌 ImportantAI Gateway is a property of a serving endpoint, not a separate deployment. Configure it once (UI tab or w.serving_endpoints.put_ai_gateway(name, ...)) and it governs every caller. The five levers: rate limiting, guardrails, usage tracking, payload logging, fallbacks.
💡 Tip · fieldAdd a gateway the moment the agent leaves your personal notebook for a shared environment. Start limits and guardrails permissive, turn on payload logging day one so you have data, then tighten using Module 13. Reuse one centrally governed gateway across agents for a single control point on cost and safety.
⚠️ GotchaThe book (B1 Ch7/8) says "MLflow AI Gateway / Databricks AI Gateway" and shows a standalone gateway URL. The current term is AI Gateway configured on a serving endpoint, with Unity AI Gateway (Beta) as the newer umbrella (MCP governance, budgets). PII detection/redaction is Preview. Exact system table and inference table names are doc details — read them back from the endpoint config rather than quoting from memory; treat live UI strings as re-check pending. Endpoint-type support: the databricks-sdk put_ai_gateway docstring notes AI Gateway is fully supported on Foundation Model, external-model, provisioned-throughput and pay-per-token endpoints, while agent endpoints (a ResponsesAgent deployed via agents.deploy) currently support only inference tables via AI Gateway — so to guardrail/rate-limit an agent, configure the Foundation Model endpoint it calls. Evolving; verify for your workspace.
