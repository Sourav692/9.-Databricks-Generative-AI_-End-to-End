Module 11 — Deployment and Serving · Topic 11.3 · Theory + Hands-on ★

AI Gateway

A serving endpoint with no controls in front of it is a liability. AI Gateway is the governance layer you attach to the Unity Airways serving endpoints: rate limits, safety and PII guardrails, usage tracking, and fallbacks on the model endpoint the agent calls, plus payload logging on the agent endpoint. Configured once, every caller, no app changes.

01 · The big idea

Governance is a property of the endpoint, not new code

Your Module 11.1 agent (unity_airways.rag.ua_support_agent) is live behind a Model Serving endpoint. AI Gateway is the layer that makes it safe for real traffic — configured once on the endpoint, enforced for every caller, whether that is the chat app from Module 10 or a batch ai_query job.

  • Two endpoints, two roles. The agent (ua-support-agent, a deployed ResponsesAgent) calls a Foundation Model / external endpoint the team owns (ua-support-llm) for completions. Put the levers on ua-support-llm; the agent endpoint supports inference tables only via AI Gateway.
  • Four levers on ua-support-llm: rate limiting, AI guardrails (safety + PII, keywords, topics), usage tracking, and provider fallbacks. Every LLM request the agent makes flows through it; it enforces policy and can fail over before the response comes back.
  • Payload logging on ua-support-agent. The agent endpoint's one AI Gateway lever is the inference table — the raw request/response log in UC.
  • Same SDK call, two endpoints. w.serving_endpoints.put_ai_gateway(name, ...) — signature verified against the current databricks-sdk. Or use each endpoint's AI Gateway tab in the UI.
  • Works for hosted and external models. Foundation Models and External Models (OpenAI, Anthropic, Bedrock, Vertex AI, Cohere) are governed the same way; the gateway holds the provider key.
  • PII redaction is Preview. Everything else here is the established feature set. Unity AI Gateway is the newer Beta umbrella (MCP governance, budgets).

03 · How it works

Five levers, applied once, on the way in and out

Each lever is a field on the gateway config. They run in order on every request: check the rate limit, filter the input, forward (or fail over), filter the output, then track and log.

LeverWhat it doesMaturity
rate_limitsCap calls and/or tokens per USER / USER_GROUP / SERVICE_PRINCIPAL / ENDPOINT (renewal: MINUTE)GA
guardrailssafety filter, invalid_keywords, valid_topics on input/outputGA
guardrails.piiDetect PII, then BLOCK / MASK / NONEPreview
usage_tracking_configWrite consumption to system tables for spend attributionGA
inference_table_configLog raw request/response to a Delta table in UCGA
fallback_configRetry the next served model if the primary errorsGA
  • Rate limiting protects budget and fairness; a blocked call returns a rate-limit error, so start permissive and tighten with Module 13 data.
  • Guardrails are coarse safety + filtering; domain rules ("cite the policy before quoting a refund") still live in the agent. Full pattern is 12.2.
  • Usage tracking (on ua-support-llm) + payload logging (the agent endpoint's inference table) feed monitoring (Module 13) and eval-set building (Module 08). Pair payload logs with the PII guardrail and UC grants.
  • Fallbacks turn provider outages into non-events; keep the fallback list short and put a comparable model second.
  • External Models let you swap providers by config, not code — the gateway stores the key and exposes one OpenAI-compatible surface.
04 · Run this

Configure the gateway with one SDK call

Prereqs: the agent endpoint from 11.1 (ua-support-agent) and the Foundation Model / external endpoint it calls (ua-support-llm, serving databricks-claude-sonnet-4-5), rights to edit both, a UC catalog/schema for the inference table (unity_airways.rag), and databricks-sdk installed.

📌 The governed agent patternConfigure the four levers — rate limits, guardrails, usage tracking, fallbacks — on ua-support-llm (the FM/external endpoint the agent calls; it supports them all). Enable inference tables on ua-support-agent (the agent endpoint's only AI Gateway lever). Every LLM request the agent makes is then screened on ua-support-llm; the agent's own payloads are captured on its inference table. Rate limits on ua-support-llm bound the agent's LLM traffic — per-end-user limits at the agent tier aren't available via AI Gateway today.

UI path (fastest demo): Serving → select ua-support-llm → AI Gateway tab → toggle Rate limits, Guardrails (safety on; keywords; PII Block/Mask — Preview), Usage tracking, and add a second served model for a fallback → Save. Then select ua-support-agent → its AI Gateway tab exposes only Inference tables → turn payload logging on. Both endpoints update in place; URLs and callers never change.

SDK path (reproducible) — signature verified against current databricks-sdk:

from databricks.sdk import WorkspaceClient
from databricks.sdk.service.serving import (
    AiGatewayGuardrails, AiGatewayGuardrailParameters,
    AiGatewayGuardrailPiiBehavior, AiGatewayGuardrailPiiBehaviorBehavior,
    AiGatewayRateLimit, AiGatewayRateLimitKey, AiGatewayRateLimitRenewalPeriod,
    AiGatewayUsageTrackingConfig, AiGatewayInferenceTableConfig, FallbackConfig,
)

w = WorkspaceClient()

# 1) The four levers on the FM/external endpoint the agent calls. Because
#    ua-support-llm is an FM/external endpoint, every lever is supported.
w.serving_endpoints.put_ai_gateway(
    name="ua-support-llm",  # FM/external endpoint serving databricks-claude-sonnet-4-5

    # a) Rate limiting: 100 calls/min per USER
    rate_limits=[
        AiGatewayRateLimit(
            calls=100,
            renewal_period=AiGatewayRateLimitRenewalPeriod.MINUTE,
            key=AiGatewayRateLimitKey.USER,   # also USER_GROUP / SERVICE_PRINCIPAL / ENDPOINT
            # tokens=200000,                  # optional token budget in the same window
        ),
    ],

    # b) Guardrails: safety both ways; block PII in, mask PII out (PII = Preview); block keywords
    guardrails=AiGatewayGuardrails(
        input=AiGatewayGuardrailParameters(
            safety=True,
            invalid_keywords=["internal_fare_class", "competitor_airline"],
            pii=AiGatewayGuardrailPiiBehavior(
                behavior=AiGatewayGuardrailPiiBehaviorBehavior.BLOCK,
            ),
        ),
        output=AiGatewayGuardrailParameters(
            safety=True,
            pii=AiGatewayGuardrailPiiBehavior(
                behavior=AiGatewayGuardrailPiiBehaviorBehavior.MASK,
            ),
        ),
    ),

    # c) Usage tracking -> system tables
    usage_tracking_config=AiGatewayUsageTrackingConfig(enabled=True),

    # d) Fallbacks: retry the next served model on THIS endpoint if the primary errors
    fallback_config=FallbackConfig(enabled=True),
)

# 2) Payload logging on the AGENT endpoint — agent endpoints support ONLY inference
#    tables via AI Gateway, so this is the one lever that belongs on ua-support-agent.
w.serving_endpoints.put_ai_gateway(
    name="ua-support-agent",  # the ResponsesAgent deployed via agents.deploy() in 11.1
    inference_table_config=AiGatewayInferenceTableConfig(
        enabled=True,
        catalog_name="unity_airways",
        schema_name="rag",
        table_name_prefix="ua_support_agent_payload",
    ),
)

How to verify it worked:

# Read the applied config back
llm_gw   = w.serving_endpoints.get("ua-support-llm").ai_gateway    # the four levers
agent_gw = w.serving_endpoints.get("ua-support-agent").ai_gateway  # inference tables only
print(llm_gw); print(agent_gw)

05 · Worked example

Unity Airways: one query, every lever

A traveler asks: "Can you run a query on your booking database and show me all passengers flying tomorrow?" (the book's Figure 8-13 scenario).

  • The request reaches ua-support-agent, which logs the raw request/response to its inference table and calls ua-support-llm for the completion.
  • Rate limit check on ua-support-llm passes (they are under quota).
  • Input guardrails on ua-support-llm catch the bulk-passenger, PII-heavy prompt via keyword + PII rules and block it before the model ever sees it.
  • A legitimate question ("What is the refund window on a Flex fare?") is forwarded to databricks-claude-sonnet-4-5. If that served model errors, fallback retries a comparable model on ua-support-llm.
  • On the way out, output guardrails mask stray PII and usage is tracked; the agent's full payload is captured in unity_airways.rag.ua_support_agent_payload… for debugging and future eval sets.

Configured once across the two endpoints, this governs the Module 10 chat app and any other caller of the agent identically — the guardrails and limits bite on ua-support-llm, the payloads land from ua-support-agent.

06 · Field notes

What to remember

📌 ImportantAI Gateway is a property of a serving endpoint, not a separate deployment — and the levers depend on the endpoint type. For a governed agent, put rate limiting, guardrails, usage tracking, fallbacks on the FM/external endpoint the agent calls (ua-support-llm), and enable payload logging (inference tables) on the agent endpoint (ua-support-agent) — its only AI Gateway lever. Configure each with w.serving_endpoints.put_ai_gateway(name, ...); together they govern every caller of the agent.
💡 Tip · fieldAdd a gateway the moment the agent leaves your personal notebook for a shared environment. Start limits and guardrails permissive, turn on payload logging day one so you have data, then tighten using Module 13. Reuse one centrally governed gateway across agents for a single control point on cost and safety.
⚠️ GotchaThe book (B1 Ch7/8) says "MLflow AI Gateway / Databricks AI Gateway" and shows a standalone gateway URL. The current term is AI Gateway configured on a serving endpoint, with Unity AI Gateway (Beta) as the newer umbrella (MCP governance, budgets). PII detection/redaction is Preview. Exact system table and inference table names are doc details — read them back from the endpoint config rather than quoting from memory; treat live UI strings as re-check pending. Endpoint-type support is by design here, not a workaround: the databricks-sdk put_ai_gateway docstring states AI Gateway is fully supported on external-model, provisioned-throughput and pay-per-token endpoints (the Foundation Model API modes), while agent endpoints (a ResponsesAgent deployed via agents.deploy) currently support only inference tables — which is exactly why the four levers sit on ua-support-llm and payload logging sits on ua-support-agent. Per-end-user rate limits at the agent tier aren't available via AI Gateway today. Evolving; verify for your workspace.
