Module 14 — AI/BI Genie · Entry 14.3 ★ · Theory + Hands-on

Curate & Tune a Genie Agent

The topic that decides whether your Genie Agent is trusted or ignored. Accuracy comes from the briefing you give it — comments, a metric view, instructions, verified answers, synonyms — tuned against a benchmark loop, not the model.

01 · The big idea

Genie's accuracy is curation, not the model

The same model gives great answers on a well-briefed Agent and garbage on a bare one. You created Unity Airways Revenue Analytics in 14.2; here you turn it into the number the CFO trusts by writing the analyst's briefing.

  • The curation stack, weakest → strongest: table/column comments → general instructions → synonyms → trusted assets (verified example SQL) → verified answers → a metric view (strongest — defines KPIs once, governed).
  • Tune with a benchmark loop, not vibes: question→expected-answer pairs, run, read the SQL, fix the miss with the right lever, re-run.
  • Everything you curate is captured in the Agent config (serialized_space) — exportable and versionable.
  • The through-line: Module 15's metric-view metadata (15.6 — synonyms, display names, formatting) is Genie curation. Define it once; Genie inherits it.
📌 ImportantFix accuracy upstream first: add table/column comments and define KPIs in the metric view before writing a single instruction. Curation compounds from a good metadata base — good metadata + a metric view + a few verified answers beats a wall of prose.
02 · The problem

A confidently-wrong number kills the project

You asked the bare Agent "total revenue last quarter by fare class." It summed only base_fare_usd, ignored ancillary_usd, and included cancelled bookings. The number is wrong — and it looks authoritative. Ask it three ways ("revenue", "sales", "how much we made") and you get three queries, sometimes three numbers. One bad number in a leadership meeting and trust is gone. Naive fixes fail:

  • "The model will figure out our schema" → it won't infer that revenue = base + ancillary from column names. Business logic isn't in the data types.
  • "One giant instruction paragraph" → prose helps but doesn't pin a query; critical KPIs need deterministic assets (verified answers + a metric view).
  • "Tune by eyeballing a few questions" → without a repeatable benchmark you can't tell a fix from churn.
  • "Curate in Genie, define metrics elsewhere" → Genie drifts from the dashboards. Curation and semantics must be one source of truth.

04 · Deep dive

Each lever, and why the order matters

1 · Comments + the metric view come first

Before a single instruction, make sure every table/key column has a COMMENT and KPIs live in bookings_metrics, not re-derived. Most wrong answers are an upstream problem — fix the metadata, then curate. The metric view carries total_revenue = SUM(base_fare_usd + ancillary_usd), cancellation_rate, and ancillary_attach_rate with the synonyms/display names from 15.6.

-- Comments are curation Genie reads as ground truth. Do this before touching instructions.
COMMENT ON TABLE unity_airways.analytics.fct_bookings
  IS 'One row per booking. base_fare_usd + ancillary_usd = revenue; status in (booked, flown, cancelled).';
COMMENT ON COLUMN unity_airways.analytics.fct_bookings.ancillary_usd
  IS 'Ancillary spend (bags, seats, upgrades) in USD. Part of total revenue.';
COMMENT ON COLUMN unity_airways.analytics.fct_bookings.status
  IS 'booked, flown, or cancelled. Exclude cancelled from revenue unless asked.';

2 · General instructions — the team's rules

"Revenue = total_revenue from bookings_metrics; exclude cancelled unless asked; 'last quarter' = previous full calendar quarter (booking_date); match tier values Blue/Silver/Gold/Platinum." Flexible and cheap, but interpreted each time — great for conventions, not a substitute for a pinned query on a critical KPI.

3 · Synonyms + column configs

Map business words to schema: "top cabin" → fare_class = 'First'; "attach rate" → ancillary_attach_rate; "churn" → cancellation_rate. Column configs add value-matching ("Gold" → the right loyalty_tier) and formatting (USD, %, dates). Most accuracy loss is vocabulary mismatch — synonyms close it directly.

4 · Trusted assets — example question → SQL

Verified query patterns Genie reuses instead of improvising: a revenue_by_fare_class example teaches the exact join, grouping, and measure usage in one shot (config: example_question_sqls, backed by join_specs + sql_snippets). More setup than instructions, far more reliable — the workhorse lever.

5 · Verified answers — certify the queries that matter

For the handful of questions leadership actually asks, mark the correct question→SQL as verified (14.5). Genie reuses that exact SQL with a trust badge — removing model variance entirely for the important numbers.

6 · The benchmark quality loop

Build 15–30 question→expected-answer pairs (real questions + edge cases). Run → read the generated SQL for each miss → diagnose the type (metadata / vocabulary / join / metric) → fix with the matching lever → re-run. This loop is tuning; every production thumbs-down becomes a new benchmark row.

05 · Worked example

Turning the wrong revenue answer into a trusted one

  1. Upstream: add COMMENTs to fct_bookings and confirm bookings_metrics defines total_revenue.
  2. Instruction: "Revenue = total_revenue; exclude cancelled; 'last quarter' = previous full calendar quarter."
  3. Synonym: "sales" / "how much we made" → total_revenue; "top cabin" → First.
  4. Trusted example: the revenue_by_fare_class SELECT grouping total_revenue by fare_class, filtered to the quarter.
  5. Benchmark: add "total revenue last quarter by fare class" with the expected number; run — it now matches.
  6. Verify: mark that question a verified answer → deterministic + badged.
  7. Loop: a week later a thumbs-down on "ancillary attach rate by tier" becomes a new benchmark row → add a synonym + example → re-run → fixed.
06 · Field notes

What to remember

📌 ImportantFix accuracy upstream first (comments + metric view), then add in-Agent curation. The metric view is the strongest lever — 15.6's synonyms/display names/formatting are Genie curation you get for free. Tune against a benchmark, not by eyeballing.
💡 Tip · fieldKeep a living benchmark set; append every production thumbs-down. Verify your top ~10 leadership questions; let the rest ride on instructions + the metric view.
