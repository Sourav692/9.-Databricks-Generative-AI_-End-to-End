Module 14 — AI/BI Genie · Topics 14.1–14.9 · Theory + Hands-on

AI/BI Genie

Ask your governed data a question in plain English. A Genie Agent turns it into SQL, runs it on a serverless warehouse, and answers with a table, a chart, and the SQL it used — over the Unity Airways Revenue Analytics schema. Its accuracy comes from curation, not the model.

01 · The big idea

A curated, governed natural-language interface to structured data

A Genie Agent (formerly "Genie Space") is a natural-language front end to Unity Catalog tables and metric views. You ask a business question; it generates SQL, runs it on a serverless SQL warehouse, and shows the answer and the SQL. The intelligence isn't the model — it's the briefing you give it.

  • Create & manage (14.1–14.2) — build the Unity Airways Revenue Analytics Agent over unity_airways.analytics: the star schema and the bookings_metrics metric view.
  • Curate & tune (14.3 ★) — instructions, trusted assets (example SQL), synonyms, a metric view, and a benchmark loop. This is where accuracy lives.
  • Trust & test (14.4–14.5) — benchmark + monitor; certify verified answers; governance is inherited from UC.
  • Go further — Agent mode multi-query research (14.6); embed + the Genie Agents API (14.7–14.8); Genie One for business users (14.9).
📌 The rule that shapes this moduleGenie is only as good as the semantic layer you point it at. Accuracy comes from curation — table/column comments, a metric view (Module 15's bookings_metrics), general instructions, verified example queries, and synonyms — not the model. Attaching the metric view is the single highest-leverage setup choice. Names: Genie Agents (not "Genie Spaces"; the REST path still says spaces), Agent mode (not "deep research"), Genie One (not "Databricks One").
02 · The problem

Self-service the business trusts, without giving up governance

Unity Airways has a clean analytics schema, but revenue managers and route planners don't write SQL. Every "quick number" becomes a ticket, and when analysts self-serve, everyone computes "revenue" differently — the same question returns three numbers in three meetings. Naive fixes break:

  • Give everyone SQL + a wiki → non-analysts won't write SQL; the wiki drifts; the bottleneck stays.
  • Point a generic text-to-SQL LLM at raw tables → no comments, no join hints, no metric definitions → confidently wrong SQL.
  • Let each team define its own metrics → that's exactly how you get three "revenue" numbers. Definitions must live once, governed.
  • Bolt on a chatbot, skip curation → the generic-LLM failure mode with a nicer logo. The product shines only when the semantic layer underneath is real.

04 · Concepts

14.1 — How Genie works

14.1 Genie Agents concepts Theory GA

A Genie Agent is a curated, governed natural-language interface to structured data. On a question it uses table/column metadata + your curation to generate SQL, runs it on a serverless warehouse, and returns the result plus the SQL it wrote — so answers are auditable, not black-box.

  • The SQL is always shown — for trust, and so a data owner can turn a good query into a verified answer.
  • Governance is inherited — UC row/column permissions, lineage, and audit apply automatically; a user only sees data they're allowed to.
  • Structured vs unstructured — Genie fronts tables/metric views; a Knowledge Assistant (Module 10) fronts documents. A Supervisor can route across both.
⚠️ Gotcha · the renameThese were "Genie Spaces" (earlier "AI/BI Genie") until 2026; docs, older blogs, and the SDK/REST path still use spaces (e.g. /api/2.0/genie/spaces/...). The product name is now Genie Agents — teach the current name, but expect the API to keep the spaces noun.
05 · Create

14.2 — Create & manage a Genie Agent

14.2 Create & manage Hands-on GA

Creating the Unity Airways Revenue Analytics Agent is a short console flow — Genie is UI-first, there's no create-agent SDK call. The work is choosing good data sources.

  • Warehouse — a running serverless (or Pro) SQL warehouse to execute queries.
  • Data sources (the important step) — attach the unity_airways.analytics star-schema tables and the bookings_metrics metric view. Prefer gold, well-commented tables + the metric view.
  • Sample questions — "total revenue last quarter by fare class?", "which routes have the highest cancellation rate?", "compare ancillary attach rate across loyalty tiers".
  • Permissions — CAN VIEW (ask) / CAN EDIT (curate + export/import for backup and dev→prod).
💡 TipThe single highest-leverage setup choice is attaching the metric view. It hands Genie the official total_revenue, cancellation_rate, and ancillary_attach_rate definitions, so it never reverse-engineers them from raw columns. Copy the Agent's space_id from its URL for the API round-trip (14.8).
06 · Cornerstone

14.3 ★ — Curate & tune (the accuracy engine)

Full deep-dive — general instructions, example SQL / trusted assets, synonyms, the benchmark quality loop, verified answers, and how each lever moves accuracy — in the Curate & tune deep-dive (curate-tune-genie.md / curate-tune-genie.html).

14.3 Curate & tune Theory Hands-on Cornerstone GA

Curation is where the Agent goes from "cute demo" to "the number the CFO trusts." You're writing the analyst's briefing, weakest to strongest:

  • General instructions — "revenue = base + ancillary; exclude cancelled; 'last quarter' = previous full calendar quarter." Cheapest, high impact.
  • Trusted assets — example SQL — verified query patterns Genie reuses instead of improvising joins.
  • Metric view (bookings_metrics) — the strongest lever: KPI logic defined once, governed, with synonyms/display names from 15.6. 15.6 metadata → 14 accuracy.
  • Synonyms — "top cabin" → First; "attach rate" → the ancillary measure; "churn" → cancellation metric.
  • Benchmarks + quality loop — question→expected-answer pairs you re-run after each change; fix misses, re-run, watch accuracy climb.
📌 ImportantTune upstream first. Most wrong answers trace to missing table/column comments or a KPI not defined in the metric view — fix those before piling on prose instructions. Curation compounds: good metadata + a metric view + a few verified answers beats a wall of instructions.
07 · Trust

14.4 + 14.5 — Test & monitor, and verified answers

14.4 Test & monitor Hands-on GA

Run your benchmark + adversarial questions before rollout; a healthy Agent asks for clarification or declines rather than inventing a join. Read the generated SQL, not just the number — a right number from wrong SQL is a latent bug. In production, Genie surfaces usage + thumbs-up/down + failing queries, and every query lands in UC audit logs. Close the loop: correct → verified answers; wrong → new instructions/examples.

14.5 Verified answers & trust/safety Theory GA

A verified answer is a certified question → SQL pair a data owner blesses; Genie reuses that exact SQL and shows a blue-check badge, removing model variance for the numbers that matter. Trust is layered: governance (UC permissions), transparency (SQL shown), determinism (verified answers + metric view), and graceful failure (clarify/decline over fabricate).

💡 Tip · fieldTreat every production thumbs-down as free labelling — each one is a user telling you exactly which question your curation doesn't yet cover. It's the highest-signal input to the 14.3 quality loop.
08 · Investigate

14.6 — Agent mode

14.6 Agent mode Theory Hands-on GA

Agent mode turns Genie from one-shot text-to-SQL into a planner: for a broad question — "Why did revenue dip last quarter?" — it breaks it into multiple sub-queries, reasons over intermediate results, and composes a cited, multi-step report (each supporting query shown). Previously called "deep research" / "Research Agent." Unity Airways: "Explain the drop in Gold-tier ancillary revenue last quarter" → it queries ancillary revenue by tier over time, isolates Gold, compares routes/fare classes, and narrates.

⚠️ GotchaAgent mode runs multiple queries — more warehouse cost + latency. Use it for genuine investigations, not "what was revenue yesterday." Verify its exact availability/label in your workspace — Genie capabilities and names change between releases.
09 · Integrate

14.7 + 14.8 — Embed, and the Genie Agents API

14.7 Embed in an external app Hands-on GA

Two paths: an iframe embed of the Genie UI in an internal portal (fastest, branded, in-context), or API integration (14.8) rendering results in your own UI — a Databricks App (Module 10) is the natural host. Either way, calls run as an identity and UC permissions still apply — embedding does not bypass governance.

14.8 Genie Agents API (Conversation API) Hands-on GA

The programmatic surface — REST (path base /api/2.0/genie/spaces/{space_id}) and the w.genie.* SDK:

# Python SDK (databricks-sdk 0.73.0) — the ergonomic path
from databricks.sdk import WorkspaceClient
w = WorkspaceClient()

# 1) start + wait (the _and_wait helper polls the message to a terminal status)
msg = w.genie.start_conversation_and_wait(space_id=SPACE_ID,
        content="What was total revenue last quarter by fare class?")

# 2) the answer is in attachments: .text (prose) and .query (generated SQL)
for att in msg.attachments or []:
    if att.query:
        print(att.query.query)                       # the SQL Genie wrote
        res = w.genie.get_message_query_result(       # 3) fetch the rows
            space_id=SPACE_ID, conversation_id=msg.conversation_id,
            message_id=msg.message_id or msg.id)
        print(res.statement_response.result.data_array)

# 4) follow-up in the same conversation (Genie keeps context)
w.genie.create_message_and_wait(space_id=SPACE_ID,
    conversation_id=msg.conversation_id, content="Now break that down by loyalty tier.")
📌 ImportantThe REST path uses spaces; the SDK namespace is w.genie — that split trips people up. Poll the message for status; fetch the query-result separately for rows. Don't invent method names — the verified surface is start_conversation(_and_wait), create_message(_and_wait), get_message, get_message_query_result (SDK 0.73.0). Full REST + SDK round-trip: notebook 14-8-genie-api.py.
10 · Business users

14.9 — Genie One

14.9 Genie One Theory Hands-on GA Domains Preview

Genie One (formerly "Databricks One") is the simplified single front door for business users — the place a non-technical stakeholder lands to ask questions across everything they're allowed to see.

  • Discovery (workspace + account level) — surfaces the Genie Agents, AI/BI dashboards, and Databricks Apps a user can access.
  • Domains (Preview) — organize discoverable assets by business area (Revenue, Operations, Loyalty).
  • Agent-mode deep research (14.6) surfaced for business users; budgets — spend controls so self-service serverless usage stays bounded.
⚠️ Gotcha · rename + maturityIt's "Genie One," not "Databricks One." The core unified UI is GA, but Domains are Preview. Verify feature-by-feature availability in the customer's workspace before promising a capability.
11 · Worked thread

Unity Airways — build it end to end

  1. Semantics first (Module 15): bookings_metrics defines total_revenue, cancellation_rate, ancillary_attach_rate once, with synonyms/display names (15.6).
  2. Create (14.2): the Unity Airways Revenue Analytics Agent on a serverless warehouse; attach the star tables + the metric view; seed the three sample questions.
  3. Curate (14.3 ★): instructions, verified example SQL, synonyms; build a small benchmark set.
  4. Test (14.4): run benchmark + edge questions; read the SQL; fix misses; re-run.
  5. Verify (14.5): mark "total revenue last quarter by fare class" as a verified answer.
  6. Investigate (14.6): Agent mode on "why did Gold-tier ancillary revenue drop?" → cited multi-query report.
  7. Integrate (14.7–14.8): drive it via the Genie Agents API (w.genie.start_conversation_and_wait → get_message_query_result), or embed via iframe.
  8. Publish (14.9): business users discover + use it through Genie One, under budgets.
