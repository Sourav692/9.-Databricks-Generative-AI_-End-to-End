What it is for

A POC scorecard turns "was the demo good?" into a small set of measurable, pre-agreed criteria with a single Go / No-go decision.

Two criteria are critical (safety and governance): if either is Not-yet, the verdict is No-go regardless of the rest. The others can be Conditional — ship the pilot with a named fix and a date.

Score the POC

—
0 / 0 criteria at Meets or better

The decision gate

  • Go — every criterion is Meets or better. Promote the pilot to production planning; run the production-readiness checklist next.
  • Conditional — a non-critical criterion is Not-yet. Ship the pilot with that gap named, owned, and dated.
  • No-go — a critical criterion (safety or governance) is Not-yet. Do not expose real users until it clears.

How each criterion is measured

  • Correctness & helpfulness — mlflow.genai.evaluate with the Correctness and Guidelines scorers against a small labeled set (Module 08).
  • Retrieval groundedness — RetrievalGroundedness and RetrievalRelevance scorers: is the answer supported by the retrieved chunks (Module 08).
  • Safety — the Safety scorer in eval plus AI Gateway guardrails on the FM endpoint the agent calls (ua-support-llm) (Modules 08, 12).
  • Latency — measured p50/p95 against a target (Module 16).
  • Cost — measured cost per 1k requests against a target (Module 16).
  • Governance — Unity Catalog grants are least-privilege and demonstrable on the model, tools, and data (Modules 00, 12).
  • User acceptance — pilot users would keep using it; qualitative feedback from the Review App.

How to use it in the room

Say this out loud
  • "We agreed these targets before we built anything, so the POC has an objective pass bar, not a vibe."
  • "Safety and governance are hard gates. Everything else can go Conditional with an owner and a date."
  • "The scores map straight to eval scorers and monitoring, so the same numbers follow us into production."
