Module 02 — Prompt Engineering · Topic 02.5 · Hands-on ★

MLflow Prompt Registry

Git for prompts. Immutable versions, mutable aliases, tags for discovery, all governed in Unity Catalog. Promote and roll back by moving a pointer, not by redeploying.

01 · The big idea

Prompts are governed assets, not notebook strings

A customer's bot promised a full refund. Nobody could say which prompt was live, who changed it, or what the last known-good version was. That is a lifecycle problem, and the Prompt Registry is the fix.

  • Prompts are named entities in Unity Catalog: catalog.schema.prompt_name.
  • Versions are immutable snapshots (edit = new version). Aliases (staging, production) are mutable pointers. Tags are version metadata.
  • The SDK is small and lives under mlflow.genai: register_prompt, load_prompt, set_prompt_alias, delete_prompt_alias, search_prompts, delete_prompt.

03 · How it works

Immutable versions, mutable aliases

register_prompt with a new name creates v1; calling it again with the same name adds v2, v3. Each version is frozen. set_prompt_alias(name, alias, version) points a label at a version and can be reassigned freely.

  • Why immutable: comparisons stay meaningful and rollback is deterministic ("go back to v3," not "hope no one overwrote it").
  • Why aliases: they turn versioned artifacts into an operational lever. Load by alias in production; re-point to promote or roll back with no redeploy.
  • Naming: a prompt name is a Unity Catalog identifier, so ownership and permissions are explicit. Keep names stable; do not encode version numbers in them.
  • Loading safely: pin a version for deterministic dev/test; wrap format() so a renamed variable fails loudly instead of producing nonsense.

Consumption pipeline in the app:

Load by alias, render variables, call the model. Re-pointing the alias changes behavior with no code change.
04 · Run this

Register, promote, load, roll back

Prereqs: mlflow[databricks]>=3.1.0; a Unity Catalog schema where you hold CREATE FUNCTION, EXECUTE, and MANAGE; the feature enabled on the workspace Previews page (Beta).

import mlflow
uc_prompt = "main.default.unity_airways_customer_support"

# v1 baseline (creates the prompt if the name is new)
v1 = mlflow.genai.register_prompt(
    name=uc_prompt,
    template="""You are a customer support assistant for Unity Airways.
Rules:
- If key details are missing, ask exactly one clarifying question.
- Do not invent fees, waivers, or exceptions.
Customer question: {{question}}
Write a concise answer (max 120 words).""",
    commit_message="v1: baseline support answer with safety and brevity constraints",
    tags={"use_case": "customer_support", "language": "en"})

# Promote via aliases (mutable pointers)
mlflow.genai.set_prompt_alias(name=uc_prompt, alias="production", version=1)

# Load by alias in the app; render variables
prompt = mlflow.genai.load_prompt(f"prompts:/{uc_prompt}@production")
content = prompt.format(question="Can I change my flight tomorrow?")

# Roll back = one line (re-point the alias)
mlflow.genai.set_prompt_alias(name=uc_prompt, alias="production", version=1)

Make the model call with the OpenAI-compatible client from the Databricks SDK:

from databricks.sdk import WorkspaceClient
w = WorkspaceClient()
client = w.serving_endpoints.get_open_ai_client()
resp = client.chat.completions.create(
    model="databricks-claude-sonnet-4-5",   # verify on the supported-models page
    messages=[{"role": "user", "content": content}],
    temperature=0.1, max_tokens=350)

How to verify it worked: the experiment's Prompts tab shows v1 with your commit message and tags; alias badges appear on the promoted versions; log prompt.name + prompt.version next to each response so any customer report traces to an exact version.

05 · Field notes

What to remember

📌 ImportantVersions are immutable, aliases are mutable. That single sentence is the whole philosophy: you never overwrite history, and you promote or roll back by moving a pointer.
💡 Tip · fieldStandardize a commit message shaped like "behavior change | context | expected outcome", a risk tag (low/medium/high), and one intentional change per version. Months-later debugging becomes a lookup instead of an investigation.
⚠️ GotchaThe book is an Early Release and the Databricks Prompt Registry is Beta. Re-verify the API surface, the mlflow[databricks] minimum version, and the UC privilege list against current docs before building a customer POC. Registry templates use {{double}} braces; convert with to_single_brace_format() for LangChain/LlamaIndex.
